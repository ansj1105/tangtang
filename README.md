### 🕹️ 탕탕특공대 스타일 게임 개발 - [likeTangTang]

### 📖 프로젝트 소개
Unity와 C#을 활용하여 **탕탕특공대(Vampire Survivors)** 스타일의 탑다운 슈팅 게임을 개발한 개인 프로젝트입니다. 게임의 핵심 재미인 **성장 시스템**과 **랜덤 스킬 선택 시스템**을 구현하는 데 중점을 두었으며, 효율적인 **데이터 관리**와 **성능 최적화**를 고려하여 설계했습니다.

개발 기간: 2025. 05. 11 ~ 2025. 06. 25

사용 기술:
  - Unity Engine
  - C#
  - JSON
  - OOP
  - 데이터 기반 설계
---

### ✨ 주요 기능 및 기술적 특징

#### 1. 데이터 기반 콘텐츠 관리 및 로비 시스템 구현
**JsonUtility**를 사용하여 몬스터, 스킬, 캐릭터, 장비 등 게임 내 모든 데이터를 **JSON** 파일로 분리하여 관리했습니다. 이를 통해 **게임 밸런스**를 코드 수정 없이 유연하게 조정할 수 있는 데이터 기반 설계를 경험했습니다.

* **메인 씬**에 **장비창, 상점, 스테이지 선택** 등 다양한 탭을 구현하여 플레이어가 게임을 시작하기 전에 충분히 준비하고 성장할 수 있는 시스템을 구축했습니다.
    * **장비 시스템**: 장비 합성, 정렬, 강화 기능을 구현하여 아이템 파밍의 재미를 더했습니다.
    * **캐릭터 시스템**: 각 스테이지 클리어 시, 다양한 캐릭터가 해방되고, 캐릭터를 강화할 수 있습니다.
    * **뽑기 시스템**: 다이아를 통한 유료 뽑기 및 **광고 시청을 통한 무료 뽑기** 시스템을 연동하여 모바일 게임의 수익화 구조에 대한 이해를 높였습니다.
    * **출석 체크 & 미션**: 출석, 미션, 업적 기능을 통해 플레이어의 지속적인 게임 접속과 플레이를 유도하는 구조를 구현했습니다.

#### 2. 핵심 게임 로직 및 최적화
* **핵심 게임 로직**: 몬스터를 처치하여 얻은 젬으로 레벨업하고, 무작위로 등장하는 스킬을 선택하여 성장하는 **로그라이트** 플레이를 구현했습니다. 스킬 레벨이 5가 된 후 특정 아이템을 획득하면 스킬이 **진화**하는 시스템을 구현하여 전략적 깊이를 더했습니다.
* **성능 최적화**: 수많은 몬스터를 동시에 생성하고 관리하기 위해 유니티의 **오브젝트 풀링(Object Pooling)** 기능을 적극적으로 활용하여 **메모리 및 CPU 부하를 최소화**했습니다.
* **스테이지 시스템**: **10개 스테이지**를 구현하고, 특정 스테이지별로 **엘리트 보스**를 배치하여 난이도를 점진적으로 상승시켰습니다. 또한, **자기장 시스템**을 구현하여 스테이지의 역동성과 긴장감을 높였습니다.

#### 3. 중앙 집중식 업데이트 시스템
유니티의 개별적인 `Update()` 호출 방식은 오브젝트 수가 많아질 때 성능 저하를 유발할 수 있습니다. 이러한 문제를 해결하기 위해 **ITickable** 인터페이스를 정의하고, 모든 업데이트 로직을 **`UpdateManager`**에서 일괄적으로 관리하는 시스템을 구축하여 **불필요한 호출 오버헤드를 줄여** 성능을 최적화했습니다.

#### 4. 드래그 앤 드랍 없는 UI 바인딩 시스템
* **문제 인식**: 기존의 유니티 UI 작업 방식은 Drag & Drop으로 컴포넌트를 연결하는 과정에서 반복적이고 비효율적인 수작업이 발생했습니다. 이는 UI 컴포넌트가 많아질수록 휴먼 에러 발생 가능성을 높이고, UI 구조 변경 시 유지보수가 매우 번거로워지는 문제를 야기했습니다.
* **해결책**: 이러한 문제를 해결하기 위해, Enum과 코드 기반의 자동 UI 바인딩 시스템을 구축했습니다. 모든 UI 스크립트가 상속받는 **`UI_Base`** 클래스를 설계하여, 유니티 에디터에서 수동으로 연결하는 대신 스크립트가 자동으로 필요한 UI 컴포넌트들을 찾아 바인딩하도록 했습니다.

**기술적 구현 상세**
* **Enum 기반 자동 바인딩**: 각 UI 프리팹 내의 컴포넌트 이름들을 Enum으로 정의했습니다. 예를 들어, `UI_Lobby.cs`에선 `enum Texts { EquipmentToggleText, ShopToggleText }`처럼 정의합니다. **`UI_Base`**의 **`Bind<T>()`** 메서드가 이 Enum 이름을 문자열로 변환하고, **`Utils.FindChild()`**를 통해 하위 오브젝트를 탐색하여 컴포넌트를 찾아 **`Dictionary<Type, Object[]>`**에 자동으로 저장합니다.
* **중앙 집중식 이벤트 관리**: **`UI_Base`**의 **`BindEvent()`** 정적 메서드를 통해 `OnClick`, `OnDrag` 등 모든 UI 이벤트를 한 곳에서 관리합니다. **`UI_EventHandler`**라는 보조 스크립트를 활용하여 이벤트 등록 코드를 일관성 있게 관리하고 재사용성을 극대화했습니다.
* **모듈화 및 재사용성**: `BindButton()`, `BindText()` 등 타입별로 바인딩 메서드를 분리하여 직관적인 코드 작성이 가능하도록 설계했습니다. 또한 팝업 애니메이션(`PopupOpenAnim`)과 같은 UI 공통 기능을 `UI_Base`에 포함시켜 재사용성을 높였습니다.
이러한 시스템을 통해 코드로 컴포넌트 간의 의존성을 명확하게 관리하고, UI의 **유지보수성**과 **확장성**을 확보할 수 있었습니다. 특히 팀 개발 환경에서도 일관된 UI 작업 방식을 유지할 수 있는 중요한 모듈이 될 것입니다.

### 🛠️ 개발 과정에서 배운 점
* **데이터 기반 설계의 중요성**: JSON을 활용하여 게임 데이터를 관리함으로써, 개발 편의성뿐만 아니라 런타임에 유연하게 콘텐츠를 변경하고 관리하는 방법을 익혔습니다.
* **성능 최적화 경험**: 오브젝트 풀링, 중앙 집중식 업데이트 시스템 등 **성능 최적화 기법**을 직접 적용하며 게임 성능을 관리하는 실무적 역량을 길렀습니다.
* **체계적인 모듈화**: `UI_Base`와 `UpdateManager`와 같은 모듈을 설계하여 컴포넌트 기반 설계에 대한 이해를 높였으며, 특히 드래그 앤 드랍 없는 UI 바인딩 시스템을 구축하여 프로젝트의 유지보수성과 확장성을 확보하는 방법을 배웠습니다.
