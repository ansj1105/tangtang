### 🕹️ 탕탕특공대 스타일 게임 개발 - [likeTangTang]

### 📖 프로젝트 소개
Unity와 C#을 활용하여 **탕탕특공대(Vampire Survivors)** 스타일의 탑다운 슈팅 게임을 개발한 개인 프로젝트입니다. 게임의 핵심 재미인 **성장 시스템**과 **랜덤 스킬 선택 시스템**을 구현하는 데 중점을 두었으며, 효율적인 **데이터 관리**와 **성능 최적화**를 고려하여 설계했습니다.

개발 기간: 2025. 05. 11 ~ 2025. 06. 25

사용 기술:
  - Unity Engine
  - C#
  - JSON
  - OOP
  - 데이터 기반 설계
---

네, 각 항목에 문제점을 명확히 추가하고, 겹치는 내용을 정리하여 포트폴리오의 완성도를 높여드리겠습니다. 항목별로 문제점과 해결책을 명시하여 논리적 흐름을 강화했습니다.

---

### **1. 데이터 기반 콘텐츠 및 성장 관리**
* **문제점**: 게임 밸런스 조정 및 콘텐츠 추가 시 코드 수정이 필요하여 개발 효율이 떨어졌습니다.
* **해결책**: **JSON 파일을 활용한 데이터 기반 설계**를 구축하여 개발자가 코드에 직접 접근하지 않고도 모든 게임 데이터(몬스터, 스킬, 장비 등)를 유연하게 관리할 수 있는 시스템을 만들었습니다. 

> 메인 씬에 장비창, 상점, 스테이지 선택 등 다양한 탭을 구현하여 플레이어가 게임을 시작하기 전에 충분히 준비하고 성장할 수 있는 시스템을 구축했습니다.
>
> * **플레이어 성장 시스템**: 장비 합성, 정렬, 강화 기능을 구현하여 아이템 파밍의 재미를 더했습니다.
> * **캐릭터 시스템**: 각 스테이지 클리어 시, 다양한 캐릭터가 해방되고 캐릭터를 강화할 수 있도록 했습니다.
> * **뽑기 시스템**: 다이아를 통한 유료 뽑기 및 광고 시청을 통한 무료 뽑기 시스템을 연동하여 모바일 게임의 수익화 구조에 대한 이해를 높였습니다.
> * **리텐션 시스템**: 출석 체크, 미션, 업적 기능을 통해 플레이어의 지속적인 게임 접속과 플레이를 유도했습니다.

---

### **2. 핵심 게임 로직 및 최적화**
* **문제점**: 수많은 몬스터와 오브젝트가 동시다발적으로 생성되고 소멸하는 환경에서 메모리 할당 및 해제 오버헤드로 인한 성능 저하가 예상되었습니다.
* **해결책**: **오브젝트 풀링(Object Pooling)**을 적극 활용하여 메모리와 CPU 부하를 최소화하고, 수백 개의 오브젝트가 한 화면에 등장하는 환경에서도 쾌적한 게임 경험을 제공했습니다.

> * **핵심 게임 로직**: 몬스터를 처치하여 얻은 젬으로 레벨업하고, 무작위로 등장하는 스킬을 선택해 성장하는 로그라이크 플레이를 구현했습니다. 스킬 레벨이 5가 된 후 특정 아이템을 획득하면 스킬이 진화하는 시스템을 구현하여 전략적 깊이를 더했습니다.
> * **스테이지 시스템**: 10개 스테이지를 구현하고, 특정 스테이지별로 엘리트 보스를 배치하여 난이도를 점진적으로 상승시켰습니다. 또한, 자기장 시스템을 구현하여 스테이지의 역동성과 긴장감을 높였습니다.

---

### **3. 중앙 집중식 업데이트 시스템**
* **문제점**: 수많은 오브젝트가 존재할 때 유니티의 개별적인 `Update()` 호출 방식은 불필요한 호출 오버헤드를 유발하여 성능 저하로 이어집니다.
* **해결책**: **`ITickable` 인터페이스**와 **`UpdateManager`**를 구축하여 모든 업데이트 로직을 일괄적으로 관리했습니다. 이로써 개별 `Update()` 호출을 하나로 통합하여 성능을 최적화할 수 있었습니다.

---

### **4. 드래그 앤 드랍 없는 UI 바인딩 시스템**
* **문제점**: 기존의 유니티 UI 작업 방식은 수동 연결로 인해 반복적인 작업, 휴먼 에러, 그리고 유지보수의 어려움을 야기했습니다.
* **해결책**: **Enum과 코드 기반의 자동 UI 바인딩 시스템**을 구축했습니다. 이를 통해 코드로 컴포넌트 간의 의존성을 명확하게 관리하고, UI의 유지보수성 및 확장성을 확보했습니다.

> * **기술적 구현 상세**
> * **Enum 기반 자동 바인딩**: 각 UI 프리팹 내의 컴포넌트 이름들을 Enum으로 정의했습니다. `UI_Base`의 `Bind<T>()` 메서드가 이 Enum을 기반으로 컴포넌트를 찾아 자동으로 연결합니다.
> * **중앙 집중식 이벤트 관리**: `UI_Base`의 `BindEvent()`를 통해 `OnClick`, `OnDrag` 등 모든 UI 이벤트를 한 곳에서 관리하여 일관성과 재사용성을 높였습니다.
> * **모듈화 및 재사용성**: `BindButton()`, `BindText()` 등 타입별로 바인딩 메서드를 분리하여 직관적인 코드 작성이 가능하도록 설계했습니다.
---

### 🛠️ 개발 과정에서 배운 점
* **데이터 기반 설계의 중요성**: JSON을 활용하여 게임 데이터를 관리함으로써, 개발 편의성뿐만 아니라 런타임에 유연하게 콘텐츠를 변경하고 관리하는 방법을 익혔습니다.
* **성능 최적화 경험**: 오브젝트 풀링, 중앙 집중식 업데이트 시스템 등 **성능 최적화 기법**을 직접 적용하며 게임 성능을 관리하는 실무적 역량을 길렀습니다.
* **체계적인 모듈화**: `UI_Base`와 `UpdateManager`와 같은 모듈을 설계하여 컴포넌트 기반 설계에 대한 이해를 높였으며, 특히 드래그 앤 드랍 없는 UI 바인딩 시스템을 구축하여 프로젝트의 유지보수성과 확장성을 확보하는 방법을 배웠습니다.
